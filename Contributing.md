# GhostConsole PRO — Contributing Guide

Thank you for your interest in contributing to GhostConsole PRO! By following these guidelines, you help keep the project organized, secure, and easy to maintain.

---

## How to Contribute

### 1. Reporting Issues

* Use the GitHub Issues tab to report bugs, feature requests, or security concerns.
* Provide detailed information including:

  * Operating system (Linux, Termux, Git Bash)
  * GhostConsole PRO version
  * Steps to reproduce the issue
  * Screenshots or logs if applicable

### 2. Feature Requests

* Describe the feature clearly, explaining why it is useful.
* Include example usage if relevant.
* Avoid duplicates by checking existing issues.

### 3. Pull Requests

1. Fork the repository and create a new branch for your work.
2. Make your changes on the branch.
3. Test thoroughly on supported platforms:

   * Linux
   * Termux (Android)
   * Git Bash (Windows)
4. Commit your changes with clear messages.
5. Submit a pull request against the `main` branch.

### 4. Coding Guidelines

* Use **Bash best practices** for scripts.
* Ensure code is **cross-platform** (Linux, Termux, Git Bash).
* Avoid using commands that require administrative privileges unless necessary.
* Keep functions modular and well-commented.

### 5. Security & Privacy

* Do **not** include passwords, tokens, or personal data in pull requests.
* Report security vulnerabilities via a GitHub Issue marked `security`.

### 6. Documentation

* Update `USAGE.md` when introducing or changing features.
* Provide clear examples for new commands or tools.

---

## Code of Conduct

All contributors are expected to follow a respectful, collaborative, and professional approach when participating.
